System Architecture

Factory Inventory Management System

A full-stack demo: a Vue 3 single-page app talking to a Python FastAPI service that serves in-memory mock data loaded from JSON files. No database — all filtering happens in Python at request time.

Vue 3 + Vite · :3000 FastAPI + Uvicorn · :8001 6 SPA views 14 REST endpoints 7 JSON datasets

Layer 03

Data Flow

The lifecycle of a single filtered request, from a user interaction to a re-rendered view.

User sets a filter in the view

A view such as Dashboard.vue binds to shared filter state from the useFilters.js composable. Changing Warehouse, Category, Status, or Time Period updates reactive refs.

Component calls the API client

On onMounted / on filter change, the view calls a method on the centralized client in client/src/api.js, e.g. api.getDashboardSummary(filters).

axios builds a query and issues HTTP GET

The client serializes active filters with URLSearchParams (skipping any set to 'all') and sends GET to http://localhost:8001/api/....

FastAPI routes & filters in memory

The matching endpoint in main.py runs apply_filters() (warehouse / category / status) and filter_by_month() (month or quarter) over the in-memory lists from mock_data.py.

Pydantic validates the response

Results are validated against response_model types (e.g. List[InventoryItem], List[Order]) and serialized to JSON. CORS is open (allow_origins=["*"]) for local dev.

View stores raw data, derives via computed

The response lands in a ref (e.g. allOrders, inventoryItems); computed properties derive totals, charts, and tables. Vue reactivity re-renders the DOM.

The Filter System

Four filters are passed as query params and applied server-side across the data endpoints.

Time Period

month — direct month (2025-03) or quarter (Q1-2025).

Warehouse

warehouse — exact match on the warehouse field.

Category

category — case-insensitive match.

Order Status

status — case-insensitive match (orders only).

Not every endpoint accepts every filter. Inventory has no time dimension, so it ignores month and status; spending and demand endpoints take no filters at all.
